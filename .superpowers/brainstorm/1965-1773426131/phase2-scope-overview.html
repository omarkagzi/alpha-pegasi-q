Phase 2 — Arboria: Scope Overview

Phase 2 has 4 major subsystems. Let's confirm the scope before diving into each.

Subsystem 1

Canvas Orchestration

Three.js ↔ Phaser 3 renderer switching. Both initialized on load. Fade transitions between orbital view and 2D game view. Neither destroyed — only hidden/paused.

Files: WorldCanvas wrapper, PlanetCanvas updates
Subsystem 2

Regional Map (2D)

Pixel-art overview map of the Arboria biome. Shows terrain, settlement markers (village/town/city), travel paths. Point-and-click navigation. Weather overlay. Day/night lighting.

Files: RegionMap, Tiled .tmj, weatherEngine
Subsystem 3

Settlement View (Phaser 3)

Top-down WASD walkable settlement. Tiled tilemap rendered by Phaser 3. Collision layers, NPC agent sprites at home locations, interaction zones, camera follow. The core playable experience.

Files: SettlementCanvas, Phaser scene, Tiled .tmj
Subsystem 4

Weather & Lighting

Biome-specific weather state machine (clear, rain, mist, overcast). Day/night cycle overlay on both regional map and settlement view. Seasonal visual changes (autumn leaves, spring blossoms).

Files: weatherEngine, seasonEngine, lighting overlays
Navigation Flow
🌍 Orbital View
Three.js → click biome → 🗺️ Regional Map
Canvas 2D or Phaser → click settlement → 🏘️ Settlement View
Phaser 3

Key Decision Point

The Regional Map sits between the orbital view and the settlement. How should we render it?

A

Phaser 3 for Both (Recommended)

Use Phaser 3 for both the regional map AND the settlement view. One engine, one canvas. The regional map is a separate Phaser scene — click a settlement marker to transition to the settlement scene. Simplest architecture, single Phaser instance serves both views.

B

HTML Canvas for Regional, Phaser for Settlement

Regional map is a lightweight HTML5 Canvas or even a React component with CSS. Only the settlement view uses Phaser 3. Lighter weight, but adds a third rendering context to manage.

C

React DOM for Regional, Phaser for Settlement

Regional map built entirely in React with styled components, SVG markers, and CSS animations for weather. Settlement view uses Phaser 3. Most "web-native" approach, easiest to style, but pixel-art aesthetic harder to achieve.
