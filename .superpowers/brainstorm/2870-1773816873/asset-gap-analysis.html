First question: What's the scope of this visual overhaul?

This determines everything — map size, character count, UI rework, sound integration.

A

Tileset Swap Only (Smallest)

Replace the placeholder tileset and character sprites with Kenney assets. Keep the existing 50×50 map layout and Tiled JSON. Rebuild the tilemap to reference the new tileset. Fastest path to visual improvement.

Pros

  • Fast — hours, not days
  • No map redesign needed
  • Minimal code changes

Cons

  • Map layout wasn't designed for these tiles
  • Won't feel like a "world" — just prettier squares
  • Doesn't deliver on PRD's "districts" or "covered markets"
B

Full Arboria Reimagining (Recommended)

Redesign Market Town from scratch using all 4 asset packs. New Tiled map with distinct districts (market square, residential, park, civic). Unique NPC sprites. Pixel-art UI for the chat overlay and HUD. Sound effects for interactions. The PRD's "welcome city" brought to life.

Pros

  • Delivers the PRD vision faithfully
  • Real character and atmosphere
  • Sets the visual standard for all future biomes
  • Uses all 4 asset packs cohesively

Cons

  • Larger scope — requires map design decisions
  • Tiled map needs full rebuild
  • Chat UI and HUD need reworking to match pixel theme
C

Full Reimagining + Phase 3 Foundation

Everything in Option B, plus: architect the map system so additional settlements and biomes can reuse the pipeline. Create a "biome tileset protocol" so when you add Kenney packs for deserts, dungeons, or oceans later, they slot in cleanly. Build the Tiled → Phaser pipeline as a reusable tool.

Pros

  • Everything from B, plus future-proofing
  • Second biome becomes much faster to build
  • Aligns with PRD's 15-biome architecture

Cons

  • Largest scope
  • Some abstraction work before visual payoff
  • Risk of over-engineering for MVP
